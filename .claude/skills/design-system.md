# Skill: Design System Enforcer

## Tokens obrigatórios
- Cores (primitivas e semânticas)
- Tipografia
- Espaçamento
- Bordas
- Sombras
- Opacidade
- Motion

## Regras
- Nunca criar estilos fora dos tokens
- Seguir nomenclatura semântica
- Garantir consistência visual
- Reutilizar componentes existentes

## Storybook
- Todo novo componente deve ser documentado
- Variantes devem estar claras
- Props tipadas

## Checklist
- [ ] Tokens respeitados?
- [ ] Nome semântico?
- [ ] Reutilizou componente?
- [ ] Documentado no Storybook?
